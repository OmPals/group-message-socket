Group Message Feed app using Flask, SocketIO and MongoDB

Join and chat with the default group


Currently using MongoDB, because it supports flexible schema.
In future, a message may consist multiple images, user id tags to reply to... etc.
In that case, designing a relational would go like...
One user to many messages, one message to many message_items (types: image, user_tag etc)
for the sake of having 1st Normal Form satisfied!
Instead, I have used MongoDB, in which I can create a collection for a chat messages' documents in a room.
Flexible schema, arrays supported for a value, Happy Designing!
MongoDB => Partition_Tolerance = True, Tune consistency and availability by specifying majority writes
